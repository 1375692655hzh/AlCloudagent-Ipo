# 招股书的"三套页码"体系（必读）

HKEX 招股书 PDF 内部同时存在三种"页码"，**互不对应**。本文件解释为什么 `--pages` 参数和章节定位需要特别处理。

## 三套页码

| 体系 | 例子 | 谁用 |
|---|---|---|
| **PDF 物理页码**（0-indexed） | `0, 1, 2, ..., 455` | pypdf / MinerU / 所有程序内部 |
| **封面/前置印刷页码**（罗马数字） | `i, ii, iii, iv, v` | LLM 看文本能识别 |
| **正文印刷页码**（阿拉伯数字） | `1, 2, 3, ..., 250` | LLM 能识别；**目录里写的就是这套** |

## 招股书典型结构

```
PDF 物理页   印刷页码   内容
─────────────────────────────────────────
0            (无)      封面
1-3          (无)      封底内页、空白
4-7          i-iv      重要提示、目录
8-12         v-ix      释义、技术词汇表
13           1         摘要开始 ← 正文从这里计 1
14           2
...
112          100       業務章节首页（目录写"業務 100 頁"）
...
362          350       最末页
363-455      I-1...    附录（用 I-1, II-1, III-1 重新编号）
```

## 关键问题

**目录里的"100 頁"指的是正文印刷页码，不是 PDF 物理页码**。

如果直接用 `100` 喂给 `--pages 99-149`（0-indexed 转换），实际会抓到 PDF 第 99-149 页，对应正文印刷页 87-137 页——**完全错位 13 页**（前置页数）。

## 偏移量定义

```
偏移量 = PDF 物理页码 − 正文印刷页码
       = 前置页数（封面 + 重要提示 + 目录 + 释义...）
```

招股书前置页数通常是 8-30 页，**每家不同**。

例：三環集團招股书前置 12 页，则：
- 目录写"業務 100 頁"
- PDF 物理页 = 100 + 12 = 112（页码 0-indexed 则是 112，因为正文第 1 页对应 PDF 物理页 12）

注意 pypdf 用 0-indexed，所以"PDF 物理页 12"在 pypdf 里是 `reader.pages[12]`。

## 三种确定偏移量的方法（按可靠性）

### 方法 A：PDF 书签（最准）

如果 PDF 带书签，`reader.outline` 直接给出每个章节对应的 PDF 物理页：

```python
from pypdf import PdfReader
reader = PdfReader("招股书.pdf")
for item in reader.outline:
    if hasattr(item, "title"):
        page = reader.get_destination_page_number(item)
        print(item.title, "→ PDF page", page)
```

输出：

```
釋義 → PDF page 8
摘要 → PDF page 12
風險因素 → PDF page 42
業務 → PDF page 112   ← 这就是"業務 100 頁"在 PDF 里的真实位置
財務資料 → PDF page 211
```

**这就是为什么层 1（书签直读）是最优路径**——不需要算偏移量，书签直接给 PDF 页。

### 方法 B：页脚投票（无书签时主路径）

招股书每页底部几乎都印有印刷页码（"– 100 –" 格式）。扫描所有页的页脚，提取印刷页码，配对 PDF 物理页，求 `(印刷页码 − PDF 物理页)` 的众数。

```python
from collections import Counter
votes = Counter()
for i, page in enumerate(reader.pages):
    text = page.extract_text()
    # 搜页脚的 "– 100 –" 格式
    m = re.search(r'[–\-—]\s*(\d{1,4})\s*[–\-—]', text[-250:])
    if m:
        printed = int(m.group(1))
        delta = printed - i
        if -5 <= delta <= 50:  # sanity
            votes[delta] += 1
offset = votes.most_common(1)[0][0]  # = 12
```

见 [`scripts/offset_calculator.py`](../scripts/offset_calculator.py) 的实现。

**可靠性**：600 页招股书扫描 200+ 个页脚投票，置信度通常 > 0.95。

**失败场景**：扫描件（图片 PDF，无文本层）、附录段（用 I-1 重新编号，偏移会变）。

### 方法 C：用户手动指定

极端情况下（扫描件、目录是图片），用户肉眼看 PDF 给出 `--pages 211-290`，工具只负责切片。

## 实战陷阱

### 陷阱 1：附录段偏移量不同

附录用 `I-1, II-1, III-1` 重新编号，正文偏移量不再适用。例如：
- 正文段："業務 100 頁" → PDF 112 页（偏移 12）
- 附录段："附錄一 I-1 頁" → PDF 363 页（偏移 363 - 0 = 363，完全不同）

当前算法（页脚投票）会把附录页脚识别失败（罗马数字 I-1 不匹配 `\d+` 正则），自动退化为只统计正文偏移。**这其实是正确行为**——定位附录时应该走层 1（书签）或层 3（手动）。

### 陷阱 2：目录跨页拼接错乱

pypdf 抽目录页文本时，表格化的目录排版会被识别成空列：

```
實際抽到：
| 業 | 務 | . | . | . | . | 100 |
| --- | --- | --- | --- | --- | --- | --- |
| 風 | 險 | 因 | 素 | . | . | 30 |
```

LLM 容错好（"業務" 和 "业务" 都能识别），但仍可能丢一些章节。这就是为什么层 2 后备方案需要"页脚投票"做独立校验——它不依赖目录解析的完整性。

### 陷阱 3：双栏排版

少数招股书目录用双栏，pypdf 抽出来左右栏顺序错乱。LLM 解析通常能恢复（保留语义），但极少数情况下章节顺序会颠倒。校验时对比 `--list` 输出和实际目录页 PDF。

## 总结

| 你要做的事 | 用什么 |
|---|---|
| 想知道章节对应的 PDF 物理页 | 层 1（书签）或 层 2（目录+偏移） |
| 想绕过页码换算的复杂性 | 层 3（`--pages`） |
| 算偏移量 | `offset_calculator.calculate_offset()` |
| 把印刷页码转 PDF 物理页 | `printed_to_pdf(printed, offset)` |

**实战建议**：先 `--probe-bookmarks` 看库内覆盖率。如果 > 60%，层 1 就够；否则层 2（需要 LLM）作为主路径，层 3 兜底。
